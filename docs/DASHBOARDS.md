# Дашборды и отчётность (для дирекции и внутренней работы)

Этот блок нужен не «для красоты», а чтобы у ОСС была измеримость:
сколько обращений, по каким темам, как быстро отвечаем, где узкие места.

## Публичная статистика (/statistics)
Показывает только агрегаты (без персональных данных):
- обращения по дням (создано/закрыто);
- распределение по направлениям.

Источник данных: таблицы `appeals_public_daily` и `appeals_public_by_direction`,
которые обновляются триггером при INSERT/UPDATE обращений.

## Внутренняя аналитика (/admin/dashboards)
Доступна руководителям направлений и руководству ОСС.
Показывает управленческие метрики:
- очередь обращений по статусам;
- нагрузка;
- динамика за период;
- (дальше) SLA и среднее время ответа.

Важно: аппарат (`staff`) не видит обращения и не может менять статусы.
