# Руководство по переводу сайта

## Структура файлов переводов

Переводы хранятся в JSON файлах в папке `frontend/nextjs/locales/`:
- `ru.json` - русский язык (основной)
- `en.json` - английский язык

## Формат файлов переводов

Файлы переводов организованы по разделам сайта:

```json
{
  "common": {
    "loading": "Загрузка...",
    "error": "Ошибка"
  },
  "navigation": {
    "home": "Главная",
    "directions": "Направления"
  },
  "appeal": {
    "title": "Подать обращение",
    "form": {
      "title": "Тема обращения",
      "description": "Описание"
    }
  }
}
```

## Как работать с переводами

### Для переводчиков

1. Откройте файл `frontend/nextjs/locales/en.json`
2. Найдите нужный раздел (например, `appeal.form`)
3. Переведите значения на английский язык
4. Сохраните файл

**Важно:**
- Сохраняйте структуру ключей (не меняйте названия ключей!)
- Переводите только значения (текст после двоеточия)
- Используйте правильную пунктуацию и форматирование
- Если текст содержит специальные символы (например, `{{name}}`), оставьте их без изменений

### Пример перевода

**Русский (ru.json):**
```json
{
  "appeal": {
    "form": {
      "title": "Тема обращения",
      "titlePlaceholder": "Кратко опишите суть вопроса"
    }
  }
}
```

**Английский (en.json):**
```json
{
  "appeal": {
    "form": {
      "title": "Appeal Title",
      "titlePlaceholder": "Briefly describe the issue"
    }
  }
}
```

## Разделы переводов

### common
Общие фразы, используемые по всему сайту:
- Кнопки (сохранить, отменить, удалить)
- Статусы (загрузка, ошибка, успех)
- Действия (поиск, фильтр, экспорт)

### navigation
Пункты меню навигации

### home
Главная страница:
- Заголовки
- Описания
- Блоки новостей и организаций

### directions
Страница направлений:
- Названия комитетов
- Описания направлений
- Кнопки действий

### appeal
Форма обращения:
- Заголовки полей
- Плейсхолдеры
- Сообщения об ошибках
- Сообщения об успехе

### status
Страница проверки статуса:
- Заголовки
- Описания статусов
- Сообщения

### admin
Админ-панель:
- Названия колонок
- Действия
- Приоритеты

### statistics, content, documents, contacts
Разделы сайта

### footer
Футер сайта

## Параметры в переводах

Если в тексте нужно подставить значение, используйте формат `{{paramName}}`:

```json
{
  "appeal": {
    "success": {
      "message": "Ваше обращение #{{id}} принято"
    }
  }
}
```

При использовании:
```typescript
t('appeal.success.message', { id: '12345' })
// Результат: "Ваше обращение #12345 принято"
```

## Добавление новых переводов

1. Добавьте ключ в `ru.json`
2. Добавьте тот же ключ в `en.json` с переводом
3. Используйте в коде через функцию `t()`:

```typescript
import { useLocale } from '../components/LocaleProvider';

function MyComponent() {
  const { t } = useLocale();
  return <h1>{t('common.siteName')}</h1>;
}
```

## Проверка переводов

После добавления переводов:

1. Убедитесь, что структура ключей совпадает в обоих файлах
2. Проверьте, что все ключи из `ru.json` есть в `en.json`
3. Проверьте работу переключения языка на сайте
4. Убедитесь, что нет синтаксических ошибок в JSON

## Часто используемые фразы

### Кнопки
- "Отправить" → "Submit"
- "Сохранить" → "Save"
- "Отмена" → "Cancel"
- "Удалить" → "Delete"
- "Редактировать" → "Edit"

### Статусы
- "Загрузка..." → "Loading..."
- "Ошибка" → "Error"
- "Успешно" → "Success"

### Формы
- "Обязательное поле" → "Required field"
- "Неверный формат" → "Invalid format"
- "Попробуйте позже" → "Please try again later"
